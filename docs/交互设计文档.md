## 一、设计原则

### 1.1 核心理念

| 原则 | 说明 |
|------|------|
| **极速优先** | 打开即记，最大程度减少操作步数 |
| **渐进呈现** | 单项目模式简单直接，多项目模式按需开启 |
| **手势自然** | 滑动、长按等原生手势与操作语义一致 |
| **反馈明确** | 每次操作都有振动/动画/Toast 反馈 |

### 1.2 操作层级

```
                    高频操作
         ┌───────────┼───────────┐
         ▼           ▼           ▼
      捕捉灵感    查看详情    标记吸收
    (输入+按钮)  (点击卡片)  (右滑/切换)
         │           │           │
         └───────────┼───────────┘
                     │
               中频操作
         ┌───────────┼───────────┐
         ▼           ▼           ▼
      删除灵感    复制内容    分享海报
    (左滑/菜单)  (长按菜单)  (详情页)
                     │
               低频操作
         ┌───────────┼───────────┐
         ▼           ▼           ▼
      批量管理    项目管理    搜索/设置
    (管理模式)   (项目卡片)  (导航栏)
```

## 三、核心交互详解

### 3.1 滑动手势（灵感卡片）

**触发区域**：灵感卡片整体区域

**手势规则**：

| 手势 | 方向 | 效果 | 反馈 |
|------|------|------|------|
| 左滑 | ← | 打开删除操作区 | light 振动 + 红色区域滑出 |
| 右滑 | → | 打开吸收操作区 | light 振动 + 绿色区域滑出 |
| 垂直滑动 | ↑↓ | 列表滚动（不触发） | 无 |

**实现细节**：
- 滑动阈值：40px
- 判断逻辑：水平位移 > 垂直位移时才触发
- 防重机制：`_slideTriggered` 标志位，单次触摸只触发一次
- 关闭机制：点击卡片其他区域 / 再次滑动回位 / 滑动另一个卡片

**状态转换**：

```
    [正常状态]
    /         \
  左滑        右滑
   ↓           ↓
[删除区打开] [吸收区打开]
   |    \    /    |
   |     关闭      |
   └──────┬───────┘
          ↓
     [正常状态]
```

### 3.2 长按操作

**灵感卡片长按**：

```
长按 500ms (不滑动)
        │
        ▼
   medium 振动反馈
        │
        ▼
  wx.showActionSheet
   ┌─────────────────┐
   │  复制内容        │  → wx.setClipboardData
   │  分享灵感卡      │  → navigateTo detail?openShare=1
   │  删除            │  → showModal → deleteInspiration
   └─────────────────┘
```

**项目卡片长按**：

```
长按 500ms
        │
        ▼
   medium 振动反馈
        │
        ▼
  wx.showActionSheet
   ┌─────────────────┐
   │  重命名          │  → showModal 输入新名称
   │  删除项目        │  → showModal 确认 → deleteProject
   └─────────────────┘
```

**防重机制**：
- 项目卡片：`_longPressFired` 标记（500ms 内去重）
- 灵感卡片：`_actionSheetOpen` 标记（关闭后解锁）
- 页面层：`_actionSheetOpen` 标记（complete 回调解锁）

### 3.3 选择模式

**进入**：首页点击「管理」按钮

**选择交互**：
```
点击卡片（选择模式下）
        │
        ├── 未选中 → 添加 localId 到 selectedIds
        │              卡片显示蓝色选中框 + ✓
        │
        └── 已选中 → 从 selectedIds 中移除
                       卡片恢复普通样式
```

**批量操作栏**：

| 按钮 | 最小选择数 | 行为 |
|------|-----------|------|
| 标记吸收 | ≥1 | 弹窗确认 → 逐条 updateInspiration(absorbed=true) |
| 转为项目 | ≥1 | 弹窗输入名称 → 创建项目 → 更新灵感 projectId |
| 批量删除 | ≥1 | 弹窗确认（红色警告）→ 逐条 deleteInspiration |

**退出**：点击「取消」/ 完成批量操作后自动退出

### 3.4 项目拖拽排序

**进入**：项目列表页点击「排序」

**拖拽交互**：
```
点击「排序」
    ↓
项目网格 → movable-area + movable-view 列表
    ↓
长按卡片上下拖动
    ↓
实时计算目标位置（y / cardHeight）
    ↓
动画"让位"：其他卡片上下移动
    ↓
松手 → 确认位置
    ↓
点击「完成」→ 持久化新顺序
```

**视觉反馈**：
- 拖拽中：被拖拽卡片轻微放大 + 阴影
- 让位动画：其他卡片平滑上下移动
- 结果：按新顺序更新 sortOrder

### 3.5 草稿续写

**保存时机**：输入内容变化后 1 秒（防抖）

**恢复时机**：页面 onLoad / onShow 时检测

**有效期限**：24 小时

```
用户输入
    ↓
1秒防抖后自动保存草稿
    ↓
意外退出 / 切换页面
    ↓
再次进入 → 检测草稿是否存在且未过期
    ↓
恢复草稿内容到输入框
    ↓
用户继续编辑 / 直接提交
```

## 四、状态与反馈

### 4.1 加载状态

| 场景 | 表现 |
|------|------|
| 首页加载 | 灵感列表骨架屏 |
| 海报生成 | 「正在生成灵感卡...」+ loading 动画 |
| 数据同步 | 「同步中...」Toast |
| 搜索 | 搜索图标旋转 |

### 4.2 空状态

| 场景 | 提示 |
|------|------|
| 无灵感（单模式） | 「还没有灵感，在上方输入框记录第一条吧」 |
| 无项目（多模式） | 「还没有项目，点击右上角 + 创建」 |
| 搜索结果为空 | 「没有找到相关灵感」 |
| 无标签 | 不显示标签云 |

### 4.3 错误状态

| 场景 | 处理 |
|------|------|
| 网络异常 | 本地正常使用，同步静默失败，下次自动重试 |
| 云函数不可用 | 降级到纯本地模式，不阻断用户操作 |
| 相册权限未授权 | 引导用户到设置页开启 |
| 图片上传失败 | Toast 提示，不阻断文本保存 |

### 4.4 成功反馈

| 操作 | 反馈 |
|------|------|
| 捕捉成功 | 「已捕捉」Toast + 列表刷新 |
| 标记吸收 | 卡片滑出 + ✓ 徽标 |
| 删除成功 | 「已删除」Toast |
| 批量操作 | 「已操作 N 条」Toast |
| 保存海报 | 「已保存到相册」Toast |
| 复制内容 | 「已复制」Toast |

### 4.5 危险操作确认

| 操作 | 弹窗内容 |
|------|---------|
| 删除灵感 | 「确定删除这条灵感？删除后无法恢复。」确认按钮红色 |
| 批量删除 | 「确定删除 N 条灵感？删除后无法恢复。」确认按钮红色 |
| 删除项目 | 「删除「项目名」后，该项目的灵感将变为无归属（不会丢失）。确认删除？」确认按钮红色 |

## 五、动效规范

### 5.1 过渡动画

| 场景 | 动效 | 时长 |
|------|------|------|
| 页面切换 | 滑动推入（navigateTo）/ 滑出返回 | 300ms |
| 卡片出现 | 从下往上淡入 + 轻微缩放 | 300ms |
| 选择模式进入 | 复选框淡入 + 底部操作栏弹出 | 250ms |
| 开关切换 | 即时切换（无动画延迟） | 0ms |
| 选中状态 | 背景色 + 边框颜色瞬时切换 | 150ms |

### 5.2 按压反馈

| 元素 | 效果 |
|------|------|
| 灵感卡片 | `transform: scale(0.98)` |
| 项目卡片 | `transform: scale(0.97)` |
| 按钮 | 背景色变深 + `transform: scale(0.96)` |
| 标签 | 背景色变深 |

### 5.3 振动反馈

| 场景 | 强度 | 时机 |
|------|------|------|
| 滑动打开操作区 | light | 滑动超过阈值 |
| 标记吸收 | medium | 标记成功 |
| 长按触发菜单 | medium | 长按超过 500ms |
| 删除确认 | heavy | （预留，暂未使用） |

## 六、边界情况处理

### 6.1 输入边界

| 情况 | 处理 |
|------|------|
| 空内容提交 | 按钮置灰不可点击 |
| 内容过长 | 详情页全文展示，卡片截断 |
| 图片超过9张 | 选择器限制 `count: 9 - 已选数` |
| 特殊字符 | 正常存储和展示 |

### 6.2 项目边界

| 情况 | 处理 |
|------|------|
| 删除最后一个项目 | 自动切换为单项目模式 |
| 单→多切换无项目 | 不自动创建默认项目，显示创建引导 |
| 项目名重复 | 允许重复（不做限制） |
| 项目为空 | 显示「0/0」统计 |

### 6.3 同步边界

| 情况 | 处理 |
|------|------|
| 未登录时操作 | 本地正常使用，登录后自动同步 |
| 版本冲突 | 云端版本严格大于本地才覆盖 |
| 网络恢复 | 自动重试同步队列 |
| 云函数超时 | 静默失败，下次重试 |

### 6.4 海报边界

| 情况 | 处理 |
|------|------|
| 无图片 | 跳过图片区绘制 |
| 无补充 | 跳过补充区绘制 |
| 内容过长 | 自动折行，限制最大行数 |
| 无位置 | 跳过位置区绘制 |
| 小程序码失败 | 跳过，不阻断海报生成 |
